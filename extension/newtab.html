Grok Mission Control — Helium New Tab

This is the extension entry point. In a full build the static Next.js output (or a local dev server) would be embedded or the page would iframe / redirect to http://localhost:3000.

The background service worker captures the previous tab's context (URL, selection, readability summary) and can inject it as the first message when the user clicks "New Mission from this page".

Current status (v0.2): Skeleton only. Load via chrome://extensions (dev mode) for testing the new-tab override.

See docs/helium-browser-agent.md (to be written) and the ACP harness for the backend contracts.
